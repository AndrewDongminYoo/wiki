## 각 사이트들의 URL 호스트 패턴

다음은 깃 저장소를 제공하는 몇 가지 사이트들의 URL 호스트 목록입니다:

- Amazon AWS CodeCommit: `aws.amazon.com`
- Assembla: `assembla.com`
- Beanstalk: `beanstalkapp.com`
- Bitbucket: `bitbucket.org`
- Codebase: `codebasehq.com`
- Fog Creek Kiln: `fogcreek.com`
- GitHub: `github.com`
- GitLab: `gitlab.com`
- Launchpad: `launchpad.net`
- Microsoft Azure DevOps: `dev.azure.com`
- Perforce: `perforce.com`
- Planio: `plan.io`

일반적으로 가장 많은 개발자들이 사용하는 깃 원격 저장소인 GitHub, GitLab, Bitbucket 모두 `.git`으로 끝나는 URL을 제공합니다. 이들 호스트에서 깃 저장소를 클론할 때는 URL의 마지막에 `.git`을 추가하여 클론할 수 있습니다. 하지만 다른 호스트에서는 그렇지 않을 수 있습니다. 항상 http URL의 마지막이 `.git`으로 끝나지는 않습니다. 따라서 깃 저장소를 클론할 때는 해당 호스트의 문서를 참조하여 올바른 URL 형식을 확인하는 것이 좋습니다.
